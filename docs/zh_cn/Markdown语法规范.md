## 标题

Markdown 支持六级标题，使用 `#` 符号表示：

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

**建议：** 文档中主标题用一级，主要章节用二级，子内容用三级或更低。

## 文本格式

### 强调方式

| 格式 | Markdown 语法 | 效果 |
|------|-------------|------|
| 普通文本 | `普通文本` | 普通文本 |
| **粗体** | `**粗体**` 或 `__粗体__` | **粗体** |
| *斜体* | `*斜体*` 或 `_斜体_` | *斜体* |
| ***加粗斜体*** | `***加粗斜体***` | ***加粗斜体*** |
| ~~删除线~~ | `~~删除线~~` | ~~删除线~~ |
| `代码` | `` `代码` `` | `代码` |

**建议：** 使用 `**` 标记粗体，使用 `*` 标记斜体，保持一致性。

- 项目 1

| 列 1 | 列 2 | 列 3 |
|------|------|------|
| 数据 1 | 数据 2 | 数据 3 |

```markdown

| 左 | 中 | 右 |
```

| 左对齐 | 居中 | 右对齐 |

## 最佳实践

### ✅ 推荐做法

1. **一致的格式**：选择一种风格并保持一致
   - 列表项使用 `-` 而不是 `*`
   - 强调使用 `**` 而不是 `__`

2. **清晰的结构**：
   - 使用适当级别的标题
   - 使用空行分隔段落和章节
   - 使用列表组织信息

3. **代码块**：
   - 总是指定语言标记（python、json、bash 等）
   - 使用适当的缩进和格式

4. **表格**：
   - 用于结构化数据比较
   - 保持列宽相对均匀

5. **链接和图片**：
   - 使用描述性的链接文本
   - 提供有意义的替代文本

### ❌ 应避免的做法

1. 过度使用不同的强调方式
2. 嵌套层级太深（建议不超过 3 层）
3. 长段落没有分段
4. 使用过多的图片
5. 不一致的语法选择

## 总结

Markdown 是一种简洁有效的文档编写方式。通过遵循本规范，可以确保文档的可读性和一致性。

**关键点：**

- 使用标题建立清晰的层级
- 使用列表组织信息
- 使用代码块展示示例
- 保持格式一致
- 适当使用表格和引用
