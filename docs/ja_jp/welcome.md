## MAHへようこそ、以下は使用ガイドです

1. 本プロジェクトはAndroidエミュレータ**のみ**をサポートしています。構成要件は以下の通りです：
    - 画面比率は16:9で、推奨（および最低）解像度は `1280x720` です。要件を満たさないことに起因する実行エラーはサポートされません。

2. 完全なMAHソフトウェアアーカイブの命名フォーマットは「MAH-プラットフォーム-アーキテクチャ-バージョン.zip」です。それ以外のものは単独で使用できない「部品」となります。よくお読みください。
ほとんどのケースにおいて、x64アーキテクチャのMAHを使用する必要があります。つまり、`MAH-win-aarch64-vXXX.zip` ではなく `MAH-win-x86_64-vXXX.zip` をダウンロードしてください。

## 機能紹介

### ゲーム起動

ゲームを起動し、メイン画面に進むまで待機します。

### ダンジョン戦闘

ダンジョン戦闘を実行します。以下のオプションが含まれます：

- ステージ選択：すべての常設ダンジョンをサポート。イベントダンジョンはサポートされていますが、一部のキーワードを手動で入力することを推奨します。

- 挑戦回数：挑戦回数を指定します。現在のDPを超過し、スタミナ回復アイテムの使用を設定していない場合、DPが尽きるまで周回します。

- スタミナ回復の使用：スタミナ回復アイテムを自動で使用します。

- チームの選択：対応するチームIDを入力します。これは各チームスロットの左上にある `team X` の数字です。

- `JSON`ファイルから内容をインポート：正規化された `JSON` ファイルから内容を読み込むことをサポートします。詳細な仕様は docs/`自動戦闘json配置.md` を参照してください。

  - `JSON`ファイルから自動取得：単一のファイルからすべての戦闘関連データを取得します。

  - カスタム組み合わせ選択：異なるファイルから戦闘関連データを取得します。

#### サポートキャラクターのカスタム選択

- 基本情報：

  1. キャラクター名（**必須**）：キャラクターの内部英語名を要求します（ほとんどが公式の英語名と同じです）。以下にいくつかの特殊な例を示します。その他の詳細は `JSON` 辞書インデックスを検索してください。検索方法はドキュメントの最後の [`関連内容の検索方法`](#関連内容の検索方法) に記載されています：
  
      - 煙の神 (Smokygod)：**the**smokygod
      - 勇者 (Hero)：**the**hero
      - 主人公 (Protagonist)：player

  2. キャラクターID（**必須**）：そのキャラクターの何枚目のカードであるかを指定します。

  3. 神器レベル（**オプション**）

  4. ARカード（**オプション**）：ARカードの内部英語名（**公式名ではありません**）を要求します。詳細は `JSON` 辞書インデックスを検索してください。[`関連内容の検索方法`](#関連内容の検索方法) を参照してください。

- サポートの絞り込み（**オプション**）：入力しない場合は、自動的に `絶対数値` が最も高いものを選択します：

  - 数値絞り込み：プレイヤーに実際に表示される数値に基づいて絞り込みます。

  - シード値絞り込み：キャラクターのシード関連の数値に基づいて絞り込みます。

- 自動編成：`JSONファイルから内容をインポート` と同様です。

- 戦闘リストの選択：`JSONファイルから内容をインポート` と同様です。

### スタミナ戦闘

ほとんどの属性はダンジョン戦闘と同じです。以下に異なるオプションを列挙します：

- ステージ選択：14章の花クエストや3つの常設デイリークエストをサポートします。イベントクエストはサポートされていますが、一部のキーワードを手動で入力することを `推奨` します。

  - カスタムデイリークエスト：デイリークエストから手動で `関連キーワード` を入力して選択します。例えば `突破界限的试炼·天（3）` のように入力します：

    1. カスタムステージ名：一部のキーワードを手動で入力することをサポートおよび `推奨` します（ステージ名：`天`）。

    2. 難易度キーワード：一部のキーワードを手動で入力することをサポートおよび `推奨` します（ステージ難易度：`（3）`）。

- スタミナ薬の使用：使用順序は `全` -> `半` -> `微`（フル -> ハーフ -> ミニ）です。

### 報酬の受け取り

ウィークリーミッションやプレゼントボックス内のアイテムを自動的に受け取ります。

## 関連内容の検索方法

**便利な検索プラグインを現在鋭意開発中です**

キャラクター名、ID、ARカード名は、`MAH/docs` 以下のインデックス辞書を参照する必要があります：

| ファイル | 用途 |
|------|------|
| [`characters.json`](#characters) | 高レアリティキャラクターインデックス |
| [`characters_lowstar.json`](#characters_lowstar) | 低レアリティキャラクターインデックス |
| [`ar.json`](#ar) | ARカードインデックス |

> ⚠️ **上記のインデックス辞書ファイルを手動で変更しないでください！**

### characters

`characters.json` では、各キャラクターの**英語名**がキーとなり、値には各カードの情報が含まれます：

| フィールド | 型 | 説明 | 可能な値 |
|------|------|------|--------|
| `rarity` | `int` | 星（レアリティ） | `1` ~ `5` |
| `element` | `str` | 属性 | `water` `fire` `wood` `world` `light` `dark` `god` `hero` `evil` `all` `zero` `infinity` |
| `weapon` | `str` | 武器 | `slash` `thrust` `knock` `snipe` `shoot` `none` `magic` `longslash` |
| `path` | `str` | リソースパス | — |

**例：**

```json
"shino": {
    "01": {
        "rarity": 3,
        "element": "evil",
        "weapon": "slash",
        "path": "assets/resource/image/character/shino/01"
    },
    "02": {
        "rarity": 5,
        "element": "evil",
        "weapon": "thrust",
        "path": "assets/resource/image/character/shino/02"
    }
}
```

### characters_lowstar

***サポートに設定されたのが低レアリティのキャラクターの場合、IDを入力しないでください***

`characters_lowstar.json` の構造は `characters.json` と似ていますが、低レアリティ（星2以下）のキャラクターは **weapon が `varies`** の場合があり、異なる属性が異なる武器に対応していることを意味します：

**例：**

```json
"pirate": {
    "rarity": 1,
    "weapon": "varies",
    "dark": {
        "weapon": "slash",
        "path": "assets/resource/image/character/pirate/dark"
    },
    "earth": {
        "weapon": "thrust",
        "path": "assets/resource/image/character/pirate/earth"
    }
},
"ouroboros": {
    "rarity": 2,
    "weapon": "none",
    "all": {
        "path": "assets/resource/image/character/ouroboros/all"
    },
    "infinity": {
        "path": "assets/resource/image/character/ouroboros/infinity"
    },
    "zero": {
        "path": "assets/resource/image/character/ouroboros/zero"
    }
}
```

> `weapon` が `varies` の場合、各属性の子オブジェクト内で個別に `weapon` を指定する必要があります。

### AR

`ar.json` では、各ARカードの**ファイル名**がキーとなります：

| フィールド | 型 | 説明 |
|------|------|------|
| `jp_rawname` | `str` | ARカードの日本語名 |
| `rarity` | `int` | 星（レアリティ）（`1` ~ `5`） |
| `tag` | `list[str]` | カードに登場するキャラクターの英語名 |
| `path` | `str` | リソースパス |

**例：**

```json
"ar_shino_moritaka.png": {
    "jp_rawname": "犬どもの戦場",
    "rarity": 4,
    "tag": ["shino", "moritaka"],
    "path": "ar\\ar_shino_moritaka.png"
}
```
